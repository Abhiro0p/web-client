Neuerungen in KomMonitor V3.0.0

Version 3 von KomMonitor bietet eine in vielen Punkten überarbeitete und erweiterte Kartenapp sowie diverse neue Konfigurationsmöglichkeiten der integrierten Geo- und Sachdaten.
  • Anpassungen in der Admin-Ansicht
    • Klassifikation pro Indikator
      Für jeden Indikator kann eine individuelle Standard-Klassifizierung festgelegt werden, inklusive Anzahl Klassen, und Klassifikationsmethode. Auch können dabei pro Raumeinheit eigene Klassengrenzen vorgegeben werden.
      Für POI Georessourcen kann ein Text anstelle eines Symbols festgelegt werden.
    • Gesamtregionale Vergleichswerte pro Indikator
      Jedem Indikator können gesamtregionale Vergleichswerte (Gesamtsumme, Durchschnittswert, räumlich nicht zuordenbare) im Metadaten-Bearbeiten Menü zugewiesen werden. Die Eingabe ist auf bereits integrierte Zeitreihenschritte begrenzt, um Datenkonsistenz zu wahren. Möglich sind manuelle Eingabe oder der Import aus einer CSV-Datei.
    • Raumeinheiten als Umringslayer
      Pro Raumeinheit kann konfiguriert werden, ob sie in der Kartenapp als Umringslayer in der Layersteuerung erscheint.
      Dies ermöglicht es, die Grenzverläufe der Raumeinheit in die Karte als Orientierungshilfe einblenden zu lassen. Dies kann nützlich sein, um bspw. punkthafte Georessourcen optisch mit Raumeinheiten-Grenzen zu überblenden.
    • Textuelle POIs (punkthafte Georessourcen)
      Punkthafte Georessourcen erlauben auch statt eines Symbols eine textuelle Unterscheidung mit bis zu drei Zeichen.
    • aufgeräumte Farbpaletten in Klassifikation
      Die verfügbaren Farbpaletten des ColorBrewer wurden auf sequentielle beschränkt. Divergierende oder qualitative Paletten wurden entfernt.
    • Fehlerbehebungen in Feature-Tabellen
      Für alle Geodatentypen wurden ein Fehler behoben, der das interaktive Editieren der Einzelzellen verhindert hat.
    • neue Konfigurationsparameter
      zur Konfiguration neuer/erweiterter Funktionalitäten wurden neue Konfigurationsparameter ergänzt. Konkret:
      • Parameter zur Steuerung, ob im Säulendiagramm eine Durchschnittslinie sowie Namen der Raumeinheiten angezeigt werden sollen // show bar chart label and average line window.__env.showBarChartLabel = false; window.__env.showBarChartAverageLine = true;
      • Für die erweiterte szenarienbasierte Erreichbarkeitsanalyse muss eine neue Software-Komponente im KomMonitor-Ökosystem konfiguriert werden. Im Client ist hierzu die URL sowie der Prozessname gesetzt werden // KomMonitor Spatial Data Processor window.__env.targetUrlToSpatialDataProcessorInstance = 'http://localhost:8090/'; window.__env.spatialDataProcessor_processName_indicatorReachabilityStatistics = "isochrone-prune";
  • Anpassungen in der Kartenapp
    • NEU - erweiterte szenarienbasierte Erreichbarkeitsanalyse inkl. Indikatoren-Statistik-Verschneidung
      die bishergie Erreichbarkeitsanalyse wurde stark ausgebaut zu einem Szenarienwerkzeug. Wesentliche Neuerungen:
      • Es können mehrere Erreichbarkeiten nacheinander angelegt und in der aktuellen Sitzung gespeichert werden.
      • Import und Export angelegter Szenarien
      • Anlegen und Bearbeiten eines Szenarios in einem eigenen Schritt-basierten Menü
      • genutzte Punktquelle kann beliebig editiert werden, um Szenarien zu erzeugen (neue Punkte setzen, vorhandene verschieben oder löschen)
      • Indikatoren-Statistik-Verschneidung: räumliche Verschneidung zwischen Einzugsgebieten und kleinräumigen Indikatoren-Statistiken für Fragestellungen wie Wie viele Personen 0-18 Jahren werden durch verfügbare Jugendhäuser erreicht/versorgt?. Beachten Sie folgende Hinweise:
        1. Berechnung der Schätzwerte erfolgt durch Überdeckung zwischen Einzugsgebieten und flächigen Indikatoren-Statistiken
        2. Ausgabe/Analyse über gesamtes Untersuchungsgebiet und Einzelpunkte inklusive PDF-Reports
        3. es werden immer nur Schätzwerte berechnet und keine Einzelpersonendaten herangezogen
        4. je kleinräumiger ein Indikator eingeht, umso realistischer werden die Schätzwerte
    • NEU - spontaner Import von Geo- oder Adressdaten
      Im neuen Datenimport-Menu (links unterhalb Georessourcen) können Daten spontan eingeladen werden, die zuvor nicht in KomMonitor eingepflegt wurden.
      Bei Adressdaten aus CSV-Dateien (aus Excel lässt sich CSV prima erzeugen) versucht KomMonitor, die Adressen in Punktkoordinaten zu übertragen (sogenannte Geokodierung aus Basis von OpenStreetMap Daten und Nominatim). Dabei können drei Fälle eintreten:
      1. genaues Ergebnis: Ort, PLZ, Straße und Hausnummer wurden vermeintlich genau gefunden
      2. ungenaues Ergebnis: Ort, PLZ passen, aber Straße zumeist ohne Hausnummer. Somit wurde eine ungenaue Koordinate bestimmt aber kein Fehler geworfen (tatsächlicher Punkt kann wenige oder viele Meter entfernt sein - je nach Straßengröße)
      3. ohne Ergebnis: Adresse konnte gar nicht verortet werden (Prüfen des Datensatzes auf Schreibfehler, doppelte oder fehlende Adresseinträge)
      Mittels geeigneter Smybolisierung sowie Funktionsbuttons erlaubt KomMonitor anschließend eine genaue Betrachtung des Imports.

      Geodaten müssen für den Import gegenwärtig im Format GeoJSON oder ESRI-Shape und im Koordinatensystem WGS84 (EPSG:4326) vorliegen. In künftigen Versionen können auch andere Koordinatensysteme und Projektionen unterstützt werden.
    • neu strukturierte Legende
      Am rechten Bildschirmrand wurde das Informations- und Legendenfenster neu angeordnet. Im obersten Menü kann wie gewohnt die Raumeinheit gewechselt werden oder der Indikatorendatensatz exportiert werden.
      Die Legende enthält nun weitere Reiter für ergänzende Indikatoren-Metadaten (ehemals verortet bei der Auswahl der Indikatoren) sowie Optionen zur spontanen Anpassung der Klassifikation über das Zahnradsymbol.
      Die Indikatoren-Statistik Funktion ist nun sichtbarer und wurde gesamtregionale Vergleichswerte erweitert (sofern diese für den aktuellen Indikator angegeben wurden)
    • Klassifizierung
      Regionaler Standard und manuelle Klassifizierung sowie spontane Wahl der Farbpalette
    • Durchschnittslinie und Label in Ranking-Diagramm
      Namen der Raumeinheiten sowie eine Querlinie des Durchschnittswerts können im Ranking-Diagramm spontan ein- und ausgeblendet werden.
    • Layersteuerung mit Transparenzregler und Raumeinheiten-Umringslayer
      Die Layersteuerung der interaktiven Karte muss nun aktiv angeklickt werden, um sie zu öffnen oder zu schließen.
      Zudem wurde sie um Transprenzregler für jeden Datensatz (Layer) erweitert.
      Sind für Raumeinheiten Umringslayer konfiguriert, so lassen sie sich via der Layersteuerung in das Kartenbild integrieren.
    • Geworessourcen POI Größe
      punkthafte Georessourcen erlauben eine einstellbare Größe (klein, mittel, groß) im oberen Teil des Georessourcen-Menus
    • Reporting
      Hochformat auswählbar
    • Fehlerbehebungen
      • bei der Schwellwertklassifizierung und der Auswahl von Georessourcen (Mehrfachselektion)
      • Export der Kartendarstellung
      • Georessourcen ohne Themenbezug lassen sich ein- und ausklappen
      • Reporting

Neuerungen in KomMonitor V2.1.0

  • Anpassungen in der Admin-Ansicht
    • Einfühurung einer Versioninfo inklusive Changelog im Begrüßungsfenster.
    • Klickbare Fortschrittsleiste in Verwaltungsmenüs. Ab sofort können die einzelnen Unterseiten eines Verwaltungsmenüs auch mittels eines Klicks auf die Fortschrittsleiste erreicht werden.
    • tabellarische Verwaltung des Zugriffsschutzes. Für Die Zugriffsrechte (lesen, editieren, löschen) bestimmter Organisationen auf einzelne Datensätze werden nun mittels einer übersichtlichen Tabelle verwaltet.
    • Aktualisierte Hinweistexte bei der Verwaltung des Zugriffsschutzes.
    • Verbesserung des POI-Geocodings. Der Import von Adressdaten als punktbasierte Georessource mittels OpenStreetMap wurde dahingehend verbessert, dass eine Bewertung der möglichen Treffer durch den Backend-Dienst vorgenommen wird. Jeder erfolgreich importierte Punkt besitzt zwei Sachattribute, geocoderank und geocodedesc mit folgenden Wertemöglichkeiten:
      1. geocoderank=2 ‐ geocodedesc = high accuracy - low risk of false positioning. feature matches same street and city/postcode and same housenumber
      2. geocoderank=1 ‐ geocodedesc = moderate accuracy - medium risk of false positioning. feature matches same street and city/postcode. but housenumber does not match
      Somit können Mehrfachtreffer bestimmter Adressen direkt geokodiert werden. Wenn ein Ergebnistreffer der Suchanfrage gleiche Straße, Hausnummer und Ort/PLZ aufweist, ist es der bestmögliche Treffer (geocoderank=2). Stimmen immerhin Ort/PLZ und Straße überein, so kann zumindest eine ungenauere Koordinate ohne richtige Hausnummer bestimmt werden (geocoderank=1).

      Adressen, die nicht geocodiert werden können führen zu einer Fehlermeldung beim Import. Solche Einträge sollten auf Tippfehler geprüft werden. Sollte auch dann kein Treffer gefunden werden, so muss diese Adresse aus der Quelldatei entfernt werden und nachträglich manuell ergänzt werden.
    • Bugfixes in Keycloak-Anbindung und Einzelfeature-Fortführen Menü
    • Sicherstellung der korrekten Integration der Abhängigkeiten uuid und jspdf

Neuerungen in KomMonitor V2.0.0

Grundlegende Erweiterung des Datenmodells und des Rollen- und Rechtemanagements

Die Aktualisierung aller KomMonitor Komponenten auf eine Version >= V2.0.0 beinhaltet bedeutende Erweiterungen im Datenmodell sowie des gezielten Zugriffsschutzes auf einzelen Ressourcen. Version >= 2.0.0 umfasst folgende Anpassungen:

  • Einführung einer Ressource Organisation. Pro angelegter Organisation werden folgende vier Rollen/Rechtestufen unterschieden
    • <orgname>-viewer ‐ lesender Zugriff auf Datensätze
    • <orgname>-editor ‐ lesender + schreibender Zugriff auf Datensätze
    • <orgname>-publisher ‐ lesender + schreibender Zugriff auf Datensätze sowie Anlegen neuer Datensätze
    • <orgname>-creator ‐ lesender + schreibender + löschender Zugriff auf Datensätze sowie Anlegen neuer Datensätze
  • Die User Verwaltung erfolgt nun verpflichtend mit der freien Software Keycloak. Dort müssen weiterhin von KomMonitor Administratoren die User verwaltet werden und mit den passenden Organisationen und Rechtestufen assoziiert werden.
  • Alle Backend Dienste sind nun vollständig über Keycloak abgesichert, sodass nur noch autorisierte Benutzer systemrelevante Aktionen wie das Antriggern von Indikatorenberechnungen oder Editieren von Konfigurationsdateien durchführen können. Dazu müssen Anpassungen in der Keycloak-Umgebung vorgenommen werden. Sprechen Sie für Unterstützung Christian Danowski-Buhren von der Hochschule Bochum an.
  • Anpassungen in der Admin-Ansicht
    • erweiterte Verwaltung von Organisationen und Rollen/Rechtestufen
    • Editierbarkeit von Feature-Tabellen. Für Indikatoren, Raumeinheiten und Georessourcen über das Feature fortführen-Menü lassen sich die einzelnen Datenzeilen in einer Feature-Tabelle anzeigen. Ein Doppelklick auf eine editierbare Zelle ermöglicht dem User nun, den Zellenwert zu ändern und diese Änderung in der Datenbank dauerhaft zu übernehmen.
    • Editierbarkeit von Georessourcen-Features. Für jeden Georessourcen Datensatz bietet das Feature fortführen-Menü eine neue Möglichkeit, Einzelobjekte hinzuzufügen (bspw. einen neuen Standort einer neuen Kita manuell hinzufügen). Dazu kann die Geometrie des Objekts mit Hilfe eines Kartenwerkzeugs erstellt werden und mittels Textfeldern können die Sachattribute des neuen Objekts gesetzt werden.
    • Geocoding-Prototyp beim Import von Adressdaten als punktbasierte Georessource. Ein neuer Backend-Dienst wurde entwickelt, der auf Basis von OpenStreetMap Daten ein Geocoding von Adressdaten umsetzt. In der Admin-Ansicht kann nun beim Import von punktbasierten Georessourcen eine Excel- oder CSV-Datei mit Adressdaten angegeben werden, sodass erfolgreich geocodierbare Objekte als POI in die KomMonitor Datenbank übernommen werden können. Das Resultat ist stark abhängig von der Qualität der in OpenStreetMap hinterlegten Daten. Der Geocodierungsdienst wird von der Hochschule Bochum betrieben und basiert auf der Software Nominatim. In künftigen Updates wird diese Mechanik weiter verbessert.
  • Anpassungen in der Kartenapp
    • Überarbeitete Reporting-Funktion in der Kartenapp. Es lassen sich nun Reporting Vorlagen auswählen, die dann mit gewünschten Metadaten und Indikatorendaten gefüllt werden können (Zeitpunkt und zeitliche Entwicklung). Außerdem können Isochronen um punktbasierte Georessourcen in einem Report erstellt werden.
  • diverse Bugfixes und Aktualisierungen einiger Programmbibliotheken.

Upgradeempfehlung KomMonitor < V2.0.0

KomMonitor wurde in Version 2.0.0 hinsichtlich des Datenmodells sowie Rechte- und Rollenkonzepts grundlegend erweitert. Es wird daher dringend empfohlen, alle Komponenten auf die neueste Version zu aktualisieren.
